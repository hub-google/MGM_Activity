# MGM (Member Get Member) 活動系統需求規格書 (SRS) - 最終確認版

## 1. 專案概述
本專案旨在建立一個 MGM 活動網站。參與者填寫基本資料產生專屬推薦連結並分享。受邀者點擊連結後，系統將追蹤點擊數，並引導受邀者回到一頁式的「活動 Landing Page」瀏覽活動細節並參加。

## 2. 系統目標與範圍
* **快速產生連結**：提供流暢的表單讓使用者註冊並取得專屬連結。
* **準確追蹤點擊**：確保每一組連結被點擊時能正確計算 (不防重複點擊，開放洗點)。
* **即時查詢與排行**：提供查詢介面，讓參與者隨時知道自己目前累積了多少點擊。
* **活動期間控制**：活動時間嚴格限制於 2026/06/20 ~ 2026/07/31。
* **資安防護**：妥善保護個資，外部無法直接存取 Google Sheets。
* **架構**：前端 GitHub Pages (網址分離架構)，後端 Google Apps Script (GAS) + Google Sheets。

## 3. 使用者角色 (User Roles)
1. **活動參與者 (分享者)**：填寫資料產生專屬連結，並可回網站查詢自己的點擊數。
2. **受邀者 (點擊者)**：點擊專屬連結進入網站，為好友加分並瀏覽 Landing Page。
3. **系統管理員 (您)**：擁有 Google Sheet 權限，管理後台資料。

## 4. 功能性需求 (Functional Requirements)

### 4.1 活動參與者 (分享者) 功能
* **F-01 填寫報名表單**：填寫「姓名」與「電話」即可報名。允許同一電話重複產生不同網址。
* **F-02 產生專屬連結**：送出後系統配發唯一識別碼 (UUID)，附加在網址後方。
* **F-03 一鍵複製**：提供「複製連結」功能。
* **F-04 查詢點擊數 (新增)**：參與者可在頁面上輸入自己的「電話號碼」，系統會回傳該號碼目前累積的總點擊次數與即時排名。

### 4.2 受邀者 (點擊者) 功能
* **F-05 記錄點擊次數與顯示好友名稱 (更新)**：受邀者點擊專屬連結進入時，系統將點擊次數加 1，並且在畫面上顯示「**您已成功為【分享者姓名】增加 1 點人氣！**」，讓體驗更有溫度。
* **F-06 網址分離架構 (更新)**：Landing Page 與產生推薦碼表單分為不同的實體網頁 (`index.html` 與 `register.html`)，各司其職，確保網址完全不同。

### 4.3 系統後端功能 (GAS API)
* **F-07 `POST /register`**：儲存參與者資料 (UUID, 姓名, 電話) 到 Sheet。
* **F-08 `POST /click`**：找到對應 UUID，點擊次數加 1，並回傳分享者的「姓名」。
* **F-09 `POST /query` (新增)**：接收電話號碼，在 Sheet 中加總該電話的所有點擊次數，並計算出目前的排名回傳給前端。
* **F-10 活動時間檢查**：所有 API 均檢查時間 (2026/06/20 ~ 2026/07/31)。

## 5. 非功能性需求 (Non-Functional Requirements)

### 5.1 資安需求 (Security)
* **S-01 個資隱私**：Google Sheet 設定為「知道連結的任何人皆可編輯」，透過後端 GAS 自動串接讀寫。GAS API 限制回傳欄位 (查詢時只回傳點擊數與排名，絕對不回傳其他人的個資)。
* **S-02 傳輸加密**：通訊強制使用 HTTPS。

### 5.2 視覺設計風格
* **V-01 設計參考**：整體風格參考 `jx3gua.com`，採用簡約、高質感的社群網站風格 (乾淨的排版、支援深/淺色模式、Noto Sans SC/Work Sans 字體)。

## 6. 資料庫結構設計 (Data Schema)
1. `UUID` (唯一識別碼，字串)
2. `Name` (姓名，字串)
3. `Phone` (電話，字串)
4. `CreatedAt` (建立時間，日期時間)
5. `ClickCount` (點擊次數，整數)
